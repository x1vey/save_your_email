Sending patterns: plain-text 1:1 = personal/transactional (lowest scores); drip 1/min mimics human MUA → non-marketing; HTML newsletter = bulk/marketing (needs higher reputation).

⚙️ 4. Infrastructure & Hedging

Hedging: using surplus capacity and reputation to absorb the cost of richer content (images, HTML, links).
Formula: capacity 100k/day at 40% open, sending only 20k/day → 80k/day headroom → can include images/links/HTML and still hold open rates.
Rule: build capacity FIRST, add content risk SECOND. Never approach capacity limits while experimenting. Never send >25% over historical max on a new day.

Bounce Rate Thresholds

< 0.5%Good
0.5–2%Acceptable — validate new additions
> 2%Validate before next send
> 5%Stop bulk sending; full audit
> 10%ESP may suspend account

5xx hard bounce → remove immediately. 4xx soft → retry up to 72h. 3+ consecutive soft → treat as hard. Ports: 587 STARTTLS / 465 implicit TLS recommended; 25 = relay only.

🗂️ 5. List Hygiene

Purchased/scraped → STOP (spam traps; violates CAN-SPAM/CASL/GDPR). Avoid role addresses: info@, admin@, sales@, webmaster@, postmaster@, noc@, abuse@, hostmaster@, support@, contact@, help@, billing@.

SegmentLast EngagementAction
Active0–30 daysAll campaigns; test new content here
Warm31–90 daysInclude with monitoring
Cold91–180 daysExclude; run re-engagement
Lapsed>180 days2–3 email re-engagement then suppress
SuppressedNeverKeep in suppression list

Spam traps: pristine (bought/scraped → immediate blacklist), recycled (old list → reputation damage), typo (no validation → minor). Validation tools: ZeroBounce, NeverBounce, BriteVerify, Kickbox.
